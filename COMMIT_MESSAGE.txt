feat: Implement complete HalalFocus system, matching algorithm, and admin panel

This commit implements the full product specification including HalalFocus
verification, matching algorithm, swipe functionality, messaging, and admin
panel features.

BREAKING CHANGES:
- Prisma schema updated with new required fields (halalScore, halalCategory,
  fullName, profileComplete, onboardingComplete)
- Database migration required: run `pnpm --filter @ummati/db exec prisma migrate dev`

Features Added:

1. HalalFocus Verification System
   - Questionnaire system for halal compliance verification
   - Automatic scoring (0-100) and categorization (halal/grey/forbidden)
   - Rejection handling for forbidden categories
   - Integrated into investor and visionary onboarding flows

2. Matching Algorithm
   - Multi-factor matching based on:
     * HalalCategory alignment (40 points)
     * HalalScore similarity (20 points)
     * Industry match (25 points)
     * Ticket size compatibility (15 points)
     * Recency bonus (0-10 points)
   - Separate recommendation engines for investors and founders
   - Filters out forbidden categories automatically

3. Swipe & Match System
   - Swipe left/right functionality
   - Automatic match creation on mutual likes
   - Match retrieval and management
   - Prevents duplicate swipes

4. Messaging System
   - Send messages within matches
   - Retrieve message history with pagination
   - Read receipt tracking
   - Unread message count

5. Admin Panel
   - View all users with halal scores and categories
   - Approve/reject users (for founders)
   - Override halalCategory with audit trail
   - Ban/unban users
   - Platform statistics and reports
   - Click-to-cycle category override

6. Updated Onboarding Flows
   - HalalFocus verification required before profile completion
   - Profile completion tracking
   - Onboarding completion status
   - Proper validation and error handling

7. API Enhancements
   - New routers: messages, admin
   - Updated routers: investorProfile, visionary, matchmaking, user
   - HalalFocus procedures in investor and visionary routers
   - Proper Clerk JWT validation in context

Database Schema Changes:
- User: added fullName field
- InvestorProfile: added halalScore, halalCategory, fullName, email, bio,
  industriesInterestedIn, investmentPreferences, profileComplete,
  onboardingComplete, halalResponses
- VisionaryProfile: added halalScore, halalCategory, fullName, email, industry,
  fundingNeeded, profileComplete, onboardingComplete, halalResponses
- New enum: HalalCategory (halal, grey, forbidden)
